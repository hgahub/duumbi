# Security Policy

## Reporting a Vulnerability

Please use GitHub's private vulnerability reporting flow for this repository:

1. Open the repository on GitHub.
2. Go to `Security` -> `Advisories`.
3. Create a private advisory with reproduction steps, impact, and suggested fix.

If private reporting is not available in your GitHub UI, open an issue with minimal detail and ask maintainers for a secure channel.

## Scope

Security reports should focus on:

- Compiler pipeline vulnerabilities
- Dependency and supply-chain issues
- Unsafe behavior in runtime or CLI handling
- Authentication or secret-handling issues in agent integrations

## Disclosure Process

- Maintainers will acknowledge the report as soon as possible.
- A fix will be prepared and validated in CI before disclosure.
- Coordinated disclosure is preferred to protect users.
